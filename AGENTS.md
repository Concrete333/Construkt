# Construkt Project Notes

Construkt is a construction payment-control product focused on escrow, approvals, release decisions, and contractor payout visibility at the work-package level.

This file is kept in the repository as a concise project-reference note.

## Product Identity

The product name is **Construkt**.

`ConstruktDev` is only the repository and workspace folder name.

## Product Scope

Construkt is intentionally narrow and focused. It is not presented as a full construction ERP, document-management suite, or general blockchain toolkit.

The product is centered on:

- project budget control
- work-package level payment governance
- milestone-aware approvals
- escrow visibility
- release readiness
- contractor payout visibility

## Public Positioning

The repository includes two complementary product surfaces:

- a static prototype that communicates the intended end-to-end user experience
- a backend-backed app that shows how the same experience maps onto real payment-control logic

Those two surfaces should be read together rather than as competing products.

## Documentation Principle

Where the prototype is ahead of the backend, the repository should describe both clearly:

- what the prototype demonstrates as product experience
- what the backend already enforces as real system logic

That distinction is part of the strength of the submission, because it shows both ambition and implementation discipline.
